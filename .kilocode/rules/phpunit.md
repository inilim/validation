# Правила для PHPUnit тестов

1. **Глубокий анализ сценариев (Critical Thinking):**
   - Не пиши «счастливые» (happy path) тесты по умолчанию.
   - Сначала анализируй код на предмет граничных значений, возможных `null`, исключений (Exceptions) и ошибок типизации (`strict_types`).
   - Тестируй логику отказа, а не только успеха.

2. **Документирование на русском:**
   - Каждому методу теста добавляй DocBlock или комментарий `//` на русском языке.
   - Формат комментария: «Проверяет [ЧТО], ожидая, что [КАК/РЕЗУЛЬТАТ]».
   - Пример: `// Проверяет выброс исключения InvalidArgumentException при передаче отрицательного баланса.`

3. **Продвинутые наборы данных (Data Providers):**
   - Вместо «экстремального объема» используй **максимальное разнообразие** данных.
   - Data Provider должен включать:
     - Граничные значения (мин/макс, +/- 1).
     - Пустые значения (empty string, null, empty array).
     - Некорректные типы данных и спецсимволы (unicode, emoji, инъекции), если это применимо.
   - Используй именованные наборы данных (array keys) в провайдерах для читаемости вывода тестов.
